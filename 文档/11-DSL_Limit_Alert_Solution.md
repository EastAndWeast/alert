# 需求分析：DSL Limit (日常结算限额) 提醒系统

## 1. 具体方案

### 技术实现路径
为了实现对单一品种每日盈亏状态（Day PnL）的精确防漏监控，系统需要在后端充当“时间序列聚合器”，因为该指标涉及历史 EOD 快照与实时数据流的差值计算。

- **后端快照**：每天零点（Server Time），自动抓取并储存所有被监控账户/产品的 **Floating PnL（开盘浮亏基准）**。
- **实时聚合**：通过 Manager API 实时监听平仓动作累加 **Closed PnL**，同时监听最新的净值获取 **Current Floating PnL**。

### 核心逻辑
1.  **事件聚合**: 不间断收集指定产品的流水。
2.  **公式计算**: 运行以下实时核算引擎：
    *   `Change in Floating Pnl = 前一天EOD时候的浮动盈亏 - 当前的浮动盈亏`
    *   `Day Pnl = -1 * Closed Pnl + Change In Floating Pnl`（注：公式中 Closed PnL 乘以 -1 通常代表从券商视角核算利润或是客户的反向净值，具体方向依赖平台结算常识）。
3.  **极值对比**:
    *   将计算出的 `Day PnL` 与配置的上限（ Max Profit Limit, 如 +5000）或下限（Max Loss Limit, 如 -10000）进行对比。
4.  **警报与截断**: 当任一方向突破阈值时，触发红色或黄色警报。

---

## 2. 例子

**场景：XAUUSD (黄金) 剧烈单边行情预警**
- **背景**: 某服务器上的黄金遭遇黑天鹅单边上扬，风控中心设置了 XAUUSD 的 `上限: +50000`，`下限: -50000` 的 DSL 保护规则。
- **动作**: 大量客户在做空黄金，导致客户账户产生巨额账面浮亏（对应如果券商做B仓则产生巨额盈利，或反之）。
- **核算**:
    *   昨日 EOD 时 XAUUSD 浮动盈亏为 `$20,000`
    *   今日平仓产生 `$15,000` 亏损。
    *   当前实时浮动盈亏被拉伸至 `-$45,000`。
    *   `Day PnL` 计算结果由于剧烈波动突然跳变为 `-$55,000`。
- **结果**:
    *   `-$55,000 < -50000`，打破底线。
    *   **警报内容**: `[DSL 结算超限预警] 品种: XAUUSD | Day PnL: -$55,000.00 | 亏损红线: -$50000.00 | 时间: 2026-03-31 16:30`

## 3. 配置参数

建议在可视化 SaaS 界面 (trading-risk-admin) 提供以下参数供风控员动态调整：

| 参数名称 | 类型 | 说明 | 示例值 |
| :--- | :--- | :--- | :--- |
| `Symbol_Filter` | Array | 监控的目标交易产品（数组，不填则默认全遍历） | `["XAUUSD", "BTCUSD"]` |
| `PnL_Upper_Limit` | Float | 盈利上限报警阈值（正数） | `5000.0` |
| `PnL_Lower_Limit` | Float | 亏损下限报警阈值（负数） | `-10000.0` |
| `Data_Source` | Enum/Array | 要监控的服务器组或白标源 | `["Server_A_Live", "Server_B_Live"]` |
| `Calculation_Timezone` | String | 判定前一天 EOD 的标准时区 | `UTC+2` |

---

## 4. 概要和触发值计算与 UI 呈现逻辑

在告警记录列表中，系统会将生硬的数字转换成富文本以提升可读性和分析效率：

### Trigger Value (触发值) 计算逻辑
- **取值**: 提取报文中的 `trigger_value` 字段。
- **呈现样式**: 使用高亮 Tag 显示。若金额为负数（亏损超限），呈现红色 `<span class="badge badge-danger">`；若金额为正数（盈利超限），呈现绿色 `<span class="badge badge-success">`。
- **拼接结构**: `<span class="badge ...">$\<value> / \<limit> (\<ratio>%)</span>`
- **示例展示**: `<span class="badge badge-danger">$-15,000 / -10k (150%) ⚠️</span>`

### Details (概要详情) 抽壳算法
- 提取报文 `details.limit` 字段，用于在卡片或表格第二列简要说明阈值。
- 后台拼接逻辑为：`Limit: $[\<details.limit>]`。
- 详情弹框内会展示详细的“盈亏防线水滴图”和限额距离刻度。
