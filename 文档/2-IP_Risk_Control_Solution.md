# 需求分析：IP 异常与黑产行为拦截系统 (Black List / Fake IP / Position Hedge)

## 1. 总体技术实现路径

基于现实业务痛点（现阶段极度依赖外包滞后查库、依靠交易员人工反应迟缓）与公司战略目标（未来打包 SaaS 商业售卖），底层的技术实现逻辑必须坚守以下铁律：

- **告别轮询，拥抱事件驱动**：废弃“外包批处理查死库”的方式，采用 **Manager API 实时事件订阅**。核心监控 MT4/MT5 的 `OnUserLogin` (防 Fake IP / Black List) 或 `OnTradeUpdate` & `OnTransaction` (防 Position Hedge)。
- **SaaS 多租户抽离**：所有的拦截逻辑必须变成可视化、动态热更新的“模板库”或“表单参数”，不能硬编码死在业务代码里。

## 2. 深入解析与功能拆解

### 规则一：Black List (黑名单库精准打击) —— [优先级：P0, MVP 目标]
- **核心逻辑**：黑产与羊毛党屡次作案，最简单的方式就是通过信誉库直接将他们“按在登录或下单的起跑线”上。
- **底层动作**：
    1. 风控操作员从网页界面添加危险目标（`Client ID` 或者 `Login/Trading IP`）。
    2. API 事件监听到新的订单或会话。
    3. 解析出的 IP/CID，与黑名单库进行高频匹配（通常采用 Redis 或内存字典做极速判定）。
- **业务展现与约束**：
    - **原因字段**：拉入黑名单的人必须填写 `Reason`（如“2025年恶意刷返佣”、“同IP恶意倒单”）。报警弹窗会直接显示该 Reason 提示给现在的当值人员，大幅消弭交接盲点。

### 规则二：Fake IP Detection (地理位置与多维特征拦截) —— [优先级：P1]
这是典型的进阶行为库校验，防的是“使用异地数据中心与购买纯净海外 IP 的黑产老手”。
- **动态基准线 (Base Location)**：抛弃单一的“国籍”比对。触发首次登录时，系统自动记录该 IP 的城市级归属地作为**“初始常驻地 (Base Location)”**。
- **强制注册地匹配 (Strict Region Matching)**：[新增能力] 支持“严格国家匹配”模式。开启后，除手动授权城市外，任何非注册地国家的 IP 登录将立即报警（防范大中华区客户使用海外 IDC 伪装）。
- **手动添加机制 (Manual Trusted Locations)**：风控员可手动录入授权账号及其合法的活动城市（格式：`账号: 城市, 国家`），解决客户报备后的合规漫游需求。
- **底层动作**：
    1. 提取 MT4/5 Client 的登录 IP -> 请求解析得到其 **ASN (运营商)** 和 **城际地理位置**。
    2. 比对客户名下已有的常驻地库（自动+手动组合库）。
- **触发判定规则**：
    - **非家宽/数据中心检测**：若解析出其网络属于 `Data Center / Hosting`（如 AWS, Vultr, 阿里云）且未被豁免，则触发报警。
    - **异地漫游检测**：若该城市的 IP 未在客户的“常驻地库”中且开启了漫游检测。
- **信任消除机制 (Trust / Whitelist)**：风控员可在报警列表点击 **『Trust Location』**，快捷将该城市添加至该客户的合法清单。

### 规则三：Cross-Account Hedge (跨账号对冲策略) —— [优先级：P1]
逻辑重构：从单纯的“同 IP 判定”升级为 **“交易对冲行为 + IP 关联诊断”**。
- **底层动作**：系统以秒级粒度扫描不同账号间是否存在交易品种相同、方向相反、手数接近的对冲单。
- **IP 关联诊断 (New)**：
    - **⚠️ 相同 IP (高风险)**：对冲双方持有完全一致的登录 IP，判定为团伙作案或多开。
    - **🔍 疑似关联 (同属地/ISP)**：即使 IP 不同，但若同属一个 **ISP (运营商)** 或 **City (城市)**，系统将标记为“疑似关联”，识别使用 4G 动态基站或同办公室的不同节点。
- **配置参数**：
    - **对冲分析时间窗口**：双方建仓间隔（如 60 秒内）。
    - **最小对手数**：过滤小微散户，仅对大于阈值（如 1.0 Lot）的潜在套利行为报警。

## 4. 概要和触发值计算与 UI 呈现逻辑

在风控系统的 Web 告警记录列表中，系统会将原始 JSON 数据转换成富文本以提升可读性和侦查效率：

### 1. Blacklist (黑名单预警)
- **触发值 (Trigger Value)**: 提取关联要素并打上标签（如：`IP: 45.33.32.156` 或 `CID: C-889923`）。使用 `badge-dark` 的硬核警示色。
- **概要 (Details)**: 直接提取黑名单命中时的记录原因（如 `[MatchCID] 恶意倒单...`）。

### 2. Fake IP (地理位置异常)
- **触发值 (Trigger Value)**: 将提取出的 `ip` 封装为一个带有地图 Emoji 的标签 `📍 165.22.x.x`。
- **概要 (Details)**:
  - 数据中心异常：计算并拼接特征如 `Datacenter: DigitalOcean`。
  - 异地漫游：拼接 `Roaming: Shanghai -> Sydney`，将当前地与基础常驻地用箭头清晰对垒出来。

### 3. Hedge IP (对冲套利异常)
- **触发值 (Trigger Value)**: 突出显示对冲发生的基础（如共享 IP 为深红色标签，或者同属地 ISP 为橙色标签）。
- **概要 (Details)**: 会动态生成“对冲博弈简报” `A(BUY) ⚡ B(SELL) | 5.0 Lots`，让审查员能够一眼看出对冲账户和对冲头寸的匹配度。而在详情弹窗中，更会绘制以时间戳为轴线的“对冲关系图”还原作案经过。
