# [2026-03-31] 部署故障修复

## 上下文
您提出线上部署报错了：`fatal: No url found for submodule path 'skill' in .gitmodules`。

## AI 执行思路
1. **原因定位**：当您让我下载 `skill` 目录（这本身属于一个独立的 GitHub repo）并放在项目根目录中时，如果您使用本地 Git 客户端将其一并拉取进去，它会在父工程（我们本项目的 Git）里面表现为一个含有单独的 Commit Hash 的 Submodule 幽灵引用。由于我们并没有正规地执行 `git submodule add` 修改 `.gitmodules`，所以任何第三方的自动化云部署服务器在执行 `git clone` 准备拉代码时，看到索引里有个幽灵 Submodule 并且没配 URL，就会发生致命错误并中止构建。
2. **解决方式**：由于 `skill` 仅仅是我在本地用于强化我的 AI 能力和预置规则的，它不是此 Web 平台本身（trading-risk-admin）需要上线的代码内容，所以我直接运用 `git rm --cached skill` 清除了该文件夹在版本库中的索引，并将其放入 `.gitignore` 避免日后再度被同步进版本库。

## 分发结果
当前已经将上述修复提交并推送到远端服务器，下一次云端构建就会成功跳过并生成干净的工作区。
