# 需求设计文档 — 告警规则克隆功能

**日期**：2026-03-27
**状态**：需求已确认，待开发

---

## 原始需求描述

现有告警规则是按数据源/服务器独立配置的。当需要将同一套规则复制到另一台服务器时，需人工逐条重建，效率低、易出错（约每周发生一次）。

用户希望实现：
- **单条规则快速克隆**：在规则详情页或列表点击克隆，选择目标服务器后完成
- **一键克隆全部规则**：将当前数据源下全部规则批量克隆到指定服务器

---

## 问题重定框

> 你说你想做**规则克隆功能**。
> 但你实际上在解决的问题是：**每次新增服务器或例行维护时，需要逐条手动重建相同的告警规则，操作繁琐且容易出错，导致员工对"新增服务器"这件事产生心理排斥。**

---

## 已确认的设计决策

| 决策点 | 已定方案 |
|--------|---------|
| **冲突处理** | 不做冲突检测，同类规则在同一服务器可并存，由用户自行管理 |
| **Group/Symbol 为空** | 空 = 代表所有，语义明确，克隆时若目标服务器不存在对应值则置空，规则仍然有效 |
| **克隆入口** | 单条和批量均使用统一的"克隆预览"弹窗（左右对比界面） |
| **权限** | 具有"规则管理"权限的管理员均可操作 |

---

## 核心 UI 交互设计

### 克隆预览弹窗（统一模式）
无论是单条克隆还是批量克隆，均弹出如下结构的预览弹窗：

```
┌────────────────────────────────────────────────────────┐
│  克隆规则到服务器                              [选择目标服务器 ▾]  │
├─────────────────────────┬──────────────────────────────┤
│    原始规则（只读）          │    克隆后预览（可编辑）          │
│                         │                              │
│  规则名称: 大单监控          │  规则名称: 大单监控 - MT5Demo  │
│  服务器: MT5_Live          │  服务器: MT5_Demo           │
│  Group: Group_A          │  Group: [空=全部]            │
│  Symbol: EURUSD          │  Symbol: [空=全部]           │
│  阈值: 100手              │  阈值: 100手                  │
│  ...                    │  ...（可直接在此编辑）            │
├─────────────────────────┴──────────────────────────────┤
│                              [取消]  [确认克隆]            │
└────────────────────────────────────────────────────────┘
```

- 右侧字段**全部可编辑**，用户可在克隆前直接调整
- 批量克隆时，弹窗展示所有待克隆规则的"卡片列表"，可展开单条查看详情

### 批量克隆额外交互
- 入口：规则列表页右上角"📋 克隆到服务器"按钮
- 点击后先选目标服务器，然后进入预览弹窗
- 弹窗顶部显示："共 N 条规则将被克隆至 [目标服务器]"
- 支持在列表中单独展开每条规则进行预调整

## 规则名称自动生成算法

克隆时系统自动生成新规则名称，规则如下：

```
格式：[原规则名/规则类型] - [目标服务器名]

示例：
- 有自定义名称："大单监控" → "大单监控 - MT5_Demo"
- 无自定义名称（用规则类型）："Large Trade" → "Large Trade - MT5_Demo"
- 目标服务器已存在同名规则：自动追加序号 → "大单监控 - MT5_Demo (2)"
```

## 三种实现路径

| 路径 | 核心思路 | 时间估算 | 主要风险 |
|------|---------|---------|---------|
| **最小路径** | 只做单条克隆 + 左右对比弹窗（不做批量） | 1-2天 | 无法满足"一键克隆全部"的核心诉求 |
| **标准路径（推荐）** | 单条 + 批量克隆，统一左右对比弹窗，自动生成规则名称 | 3-5天 | 批量时弹窗的交互细节需打磨 |
| **完整路径** | 以上 + "规则模板库"，保存模板跨服务器复用 | 2-3周 | 模板管理本身会成为新的维护负担 |

**推荐：标准路径**

## CEO 评审意见 — 2026-03-27

**选择模式**：✅ 保持范围（核心范围合理，打磨体验细节）

**评审摘要**：功能方向正确，批量克隆的核心链路逻辑清晰。最大的体验问题是：弹窗假设用户需要"逐条审查"，实际上大多数用户只想"全部原样克隆"，强制展开预览反而是摩擦，需要提供快速模式。

**需要调整的方案内容**：

- **[必须] 批量预览弹窗增加"快速 / 详细"模式切换**
  - 默认进入快速模式：折叠所有卡片，只显示规则名列表 + 全局启用/禁用开关
  - 用户可点击"展开详情"进入详细模式，才显示左右对比
  - 避免强迫所有用户逐条审查

- **[必须] 处理"无目标服务器"的空状态**
  - 当前账号下只有一台服务器时，克隆按钮灰显
  - Hover 提示："暂无其他服务器可选，请先添加数据源"

- **[加分] Toast 成功提示带跳转快捷入口**
  - 克隆成功后：「✅ 成功克隆 N 条规则 | 查看目标服务器规则 →」
  - 用户无需手动切换导航来验证结果

**关键押注**：

- **[押注 1]** 大多数用户批量克隆时不需要逐条预览，"快速确认"是主路径。如果错误，信号是用户频繁切换详细模式调整字段。
- **[押注 2]** "输入目标服务器名"的二重确认不会造成摩擦，用户会认为是合理保护。如果错误，信号是用户抱怨或复制粘贴方式绕过。

**评审结论**：✅ 批准开发，前提是落实以上两个必须修改点（快速模式 + 空状态处理）
