# 敞口告警去除“最大提醒次数”修改成果

## 变更内容简述
根据需求，本次更新完全移除了敞口告警（Exposure Alert）中的“最大提醒次数”设置，涵盖：
1. `mock-data.js` 数据层：清理了模拟告警规则对象（R040, R041）中残留的 `max_remind_count` 属性。
2. `rules.js` 视图层：清理了规则卡片（Card View）中的对应字段渲染。
3. `rules.js` 交互层：删除了“新增/编辑”敞口告警弹窗表单对应的输入框及提取 FormData 时对应的获取逻辑。
4. `i18n.js` 语言包：删除了多语言对象中的对应 key (`max_remind_count_label`)。

## 测试步骤与结果
因为是前端纯视图及其数据驱动逻辑的调整，可通过刷新 Web 应用来体验：
1. **现有规则卡片检查**:
   - 敞口告警对应的列表项中不再出现原有的“最大提醒次数：1 次”。
2. **新增/修改表单检查**:
   - 点击“新增敞口告警”，在暴露数值、时间间隔等参数下方，不再出现“最大提醒次数”输入框。
3. **数据保存与流转检查**:
   - 将敞口保存后审查对象或执行规则功能，前端不再报错，数据体中也不再携带此字段。
