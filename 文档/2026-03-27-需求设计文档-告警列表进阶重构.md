# 需求设计文档 — 告警列表进阶重构

> 文档日期：2026-03-27
> 状态：需求梳理完成（待 /架构评审）

---

## 一、原始需求描述
1. 新增规则名称的搜索。
2. 新增规则名称在字段列表的显示。
3. 新增右上角设置：可以配置哪些字段在列表显示，列表显示的顺序。
4. 新增升序降序功能（对于可以排序的字段）。
5. 优化显示方式：左右滑动，部分重要内容固定。

---

## 二、问题重定框

你说你想做**“一系列高级表格控件的交互机制”**。但你实际上在解决的问题是：
> **“随着风控维度的细化与字段的暴增，过去原始初级的纯静态换行表格在主流屏幕上已引发了视觉灾难。高频接警的审查员在信息过载面前，极度需要一个兼具结构锁定、视觉焦点定制（显隐）以及关键追踪（自定义名称查询）的专业级数据驾驶舱。”**

---

## 三、关键假设与挑战

- **[假设 1]**：你假设允许用户随意配置“显隐和顺序”是最好的体验。
  - **[挑战]**：实际的生产环境中，某些系统基要元数据（如 `告警ID`、`状态`、`操作`）如果被隐藏或乱序，将直接导致工单处理中断。**重构时是否应该将这些关键字段列为“强制呈现且位置冻结（Sticky）”，只允许用户调整次要数据列？**
- **[假设 2]**：你假设实现这些高级桌面表格功能就是直接堆代码。
  - **[挑战]**：手撸带拖拽、动态重排、双向滚动及固定列头的 Table 是前端灾难级的天坑（极易出现 CSS 事件穿透与边框塌陷）。**是选择引入如 `DataTables.js` 一样笨重但现成的工业级第三方库（这会破坏极简原则且需要重写样式），还是通过 HTML5 原生拖拽 API + `position: sticky` 加上 `localStorage` 手搓一套轻量可控的防御性组件？**

---

## 四、三种实现路径

| 路径 | 核心思路 | 时间估算 | 主要风险 |
|------|---------|---------|---------|
| 最小路径（验证假设） | 不做自由定制。单纯加入横向滚动 + 两侧固定，追加新列与防呆搜索框。 | 极低 | 没有真正解决用户千人千面的视图偏好问题。 |
| **标准路径（坚守极简）** | 坚持现有的原生轻量路线。手撕一个弹窗：内置 Checkbox 控显隐，使用箭头或拖拽排次序。数据层存入 `localStorage`，配合 CSS Sticky 呈现超长横向冻结架构。 | 中等 | 原生事件的边界处理琐碎，但代码零外部依赖，极度清爽安全。 |
| 完整路径（推翻重构） | 整体废弃当前 HTML 渲染规则，全盘引入第三方企业表格库（如 `ag-Grid`），以获得像素级完美。 | 极长 | 项目急剧膨胀，与您追求零摩擦的愿景背道而驰。 |

---

## 五、推荐方案

秉承着项目至此的精益作息，强烈推荐 **标准路径（坚守极简）**：
1. **纯净加持**：不破坏原本写好的 `alerts.js` 数据读取流，仅是对渲染 `<tr>` 的拼接法进行干预（使用循环数组读取）。
2. **两级冻结**：最左侧冻结（`告警ID`, `所属公司`...），最右侧冻结（`状态`, `操作`），中间一堆参数支持 `overflow-x` 穿梭滑动。
3. **设置齿轮面板**：在导出按钮旁加一个齿轮，弹出全视觉化的偏好选择器。
