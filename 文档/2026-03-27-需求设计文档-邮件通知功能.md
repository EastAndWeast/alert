# 需求设计文档 — 邮件通知功能

> 文档日期：2026-03-27
> 状态：需求梳理完成（待 /架构评审）

---

## 一、问题重定框

> 现有通知渠道（Telegram、Lark、Teams、Slack）配置门槛高（Webhook配置、Bot Token等技术名词），**每次新客户入驻必须依赖技术人员介入**，无法自助完成。
>
> 邮件是用户天然熟悉的渠道，**通过"账号级别"绑定（而非规则级别配置），最大化降低客户使用门槛**。

---

## 二、核心架构决策

### 🔑 关键设计：邮件对齐 Web 渠道模式（而非 Telegram/Lark 模式）

| 对比项 | Telegram / Lark | **邮件（Email）** |
|--------|----------------|-----------------|
| 推送目标 | 规则级别配置目标群组 | **账号级别**：每个账号绑定自己的通知邮箱 |
| 开关控制 | 配置 Webhook/Bot | 账号自己选择 ON/OFF |
| 多人收件 | 配置群组天然多人 | 所有"开启邮件"的账号各自收 |
| 配置复杂度 | 高（技术门槛） | **低（填邮箱即可）** |

> **结论**：邮件不在规则层面配置目标，而是每个账号绑定通知邮箱并控制开关，规则触发时广播给所有已开启邮件的账号。

---

## 三、功能模块详细设计

### 模块 A：Global 发件服务管理（Global 管理员）

**入口**：系统设置 → 发件服务管理（Email Services）

**功能说明：**
- **新增/编辑/删除**发件服务配置
- 每条配置包含：服务名称、服务商类型（预设模板或自定义SMTP）、认证信息
- **分配给子公司**：每个公司账号可配置"使用哪个发件服务"

**预设服务商模板（海外优先）：**

| 服务商 | 类型 | 主要参数 |
|--------|------|---------|
| SendGrid | API | API Key |
| Mailgun | API | API Key + Domain |
| AWS SES | API | Access Key + Secret + Region |
| Postmark | API | Server API Token |
| Mailchimp Transactional (Mandrill) | API | API Key |
| 自定义 SMTP | SMTP | Host / Port / TLS / Username / Password |

---

### 模块 B：公司管理 → 邮件服务分配（Global 管理员）

**入口**：公司管理（Companies）→ 编辑某公司 → 邮件配置

**功能说明：**
- 开启/关闭该公司的邮件推送功能
- 选择该公司使用哪个发件服务（从模块A的列表选择）
- 配置该公司的发件人名称（From Name）和发件邮箱（From Email）

---

### 模块 C：账号通知邮箱（子公司管理员 / 用户本人）

**入口 1**：创建账号时 → 通知邮箱字段（默认填入注册邮箱，可修改）

**入口 2**：账号设置 → 通知偏好 → 邮件通知邮箱（可修改）

**功能说明：**
- 每个账号绑定一个**通知邮箱**（≠ 登录邮箱，但默认一致）
- 账号可选择是否开启邮件通知（ON/OFF 开关）
- 可发送**测试邮件**，验证配置是否正确

---

### 模块 D：全局自动推送关联（无需前端配置）

**触发机制**：系统任意风控告警规则被触发时

**功能说明：**
- **取消**在各个风控规则配置面板内单独勾选“是否从 Email 渠道推送”的冗余配置。
- 邮件作为**全局通知基础设施**，只要：(1) 所属公司配置了发件服务 (2) 用户独立开启了接收告警，该用户就会按权限范围自动收到产生的告警邮件。
- 采用最极致的防呆极简设计，避免陷入复杂交叉的多对多规则配置泥潭。

---

## 四、用户权限对照表

| 角色 | 能做什么 |
|------|---------|
| **Global 管理员** | 创建/管理多个发件服务配置；为每个子公司分配发件服务；控制子公司邮件功能开关 |
| **子公司管理员** | 管理该公司下账号的通知邮箱；为规则配置是否启用 Email 推送 |
| **子公司普通账号** | 修改自己的通知邮箱；开启/关闭自己的邮件通知接收 |

---

## 五、推荐实现范围（标准路径）

```
✅ Global 端：
   - 发件服务管理页（CRUD + 海外服务商预设模板）
   - 公司管理中增加邮件服务分配字段

✅ 账号端：
   - 创建账号时新增"通知邮箱"字段
   - 账号设置中新增邮件开关 + 通知邮箱修改 + 测试发送

✅ 规则端：
   - 告警触发引擎默认根据前置依赖进行全局推送扫描（无需新增UI配置项）

❌ 暂不做：
   - 邮件发送状态追踪（已送达/退信）
   - 按邮件量计费
   - 退订链接（合规）
```
