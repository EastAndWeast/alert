# AI 沟通记录 - 告警列表进阶重构与全链路功能实现

**日期**：2026-03-27
**参与者**：人类用户 & AI 助手 (Antigravity)

## 1. 核心需求概览
本次沟通主要围绕告警系统的三大核心升级展开：
- **邮件通知全链路**：要求在公司列表和用户列表直观查看邮件服务是否开启，并增加防呆机制。
- **规则名称 (Custom Name)**：在告警规则中增加自定义名称字段，并在告警记录中优先展示。
- **告警列表进阶重构**：实现高性能表格布局，支持横向滚动、列冻结、自定义列显隐、搜索与排序。

## 2. 关键沟通节点与决策

### 2.1 邮件配置可见性
- **决策**：在公司列表增加“邮件服务”状态列；在用户列表增加“邮件通知”状态标签。
- **防呆**：若公司未配置邮件服务，则用户编辑界面禁用邮件开关并提供明确提示。

### 2.2 规则名称展示逻辑
- **决策**：在 `rules.js` 和 `alerts.js` 中实现“自定义名称优先”逻辑。如果设置了自定义名称，列表和详单均以此名称为准；若未设置，则回退到原始规则类型名称。

### 2.3 告警列表 UI 架构 (核心难点)
- **挑战**：表格列数显著增加，传统布局容易造成内容堆叠或布局坍塌。
- **决策**：采用原生 CSS `sticky` 方案。
  - **左固定**：告警ID。
  - **右固定**：操作按钮。
  - **中间区**：横向自由滚动。
- **迭代修复**：在实施过程中遇到了 SPA 单例内存残留导致的内容重叠，以及亮色主题下固定列黑色背景的问题。最终通过强制重置 `this.columns` 和优化 CSS 变量背景色逻辑解决。
